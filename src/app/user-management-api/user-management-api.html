< User Management API />

Overview

A RESTful API built with Laravel for user authentication, authorization and account management. The project demonstrates secure authentication, role-based access control and API development best practices.

Laravel 12 PHP REST API Authentication Authorization Middleware MySQL

Key Features

  • User registration
  • User login/logout
  • Token-based authentication
  • Role-based authorization
  • User CRUD operations
  • Request validation
  • Password hashing
  • Soft deletes

Technical Highlights

1. User Registration

Purpose

Creates new user accounts after validating the incoming data.

User Input
↓
Validation
↓
Password Hashing
↓

2. User Authentication (Login)

Purpose

Validates user credentials, verifies the password against the stored hash and generates a Sanctum access token for authenticated API requests.

Email & Password
↓
Find User
↓
Verify Password
↓
Generate Sanctum Token
↓

3. Protected Routes

Purpose

Protects sensitive endpoints using Laravel Sanctum authentication, ensuring that only authenticated users can access restricted resources.

API Request
↓
Sanctum Token
↓
Authenticate User
↓
Access Protected Route

4. Role-Based Authorization

Purpose

Uses custom middleware to verify user roles and restrict access to endpoints based on permissions.

Authenticated User
↓
Role Middleware
↓
Check User Role
↓
Access Granted / Forbidden

API Endpoints

Method Endpoint Access Description
POST /register Public Register a new user account.
POST /login Public Authenticate a user and generate an access token.
POST /logout Authenticated Revoke the current access token.
GET /users Admin Retrieve all users.
GET /users/{id} Admin Retrieve a specific user.
POST /users Admin Create a new user.
PUT /users/{id} Admin Update user information.
DELETE /users/{id} Admin Soft delete a user account.
