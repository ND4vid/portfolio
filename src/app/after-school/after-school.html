Overview

A backend system for a Unity-based game that securely receives, stores, and manages player statistics through a RESTful API. This project demonstrates client-server communication, and persistent gameplay data management using Laravel.

Unity C# MySQL Statistics REST API Client-Server

Key Features

  • Calculating the in-game steps made by the current player based on stepdistance
  • Converting the recorded data to steps and storing it in a local variable

The Method

Purpose

Tracks the player's movement by accumulating traveled distance between frames. When a certain distance threshold is reached, it increments the step counter.

The method also triggers synchronized footstep audio after each step.

↓

Method Call

Purpose

The method is executed every frame from Update(), continuously tracking adn updateing steps in real time.

API Workflow

1. API Configuration

Purpose

2. Step Counter Initialization

Purpose

Initializes the variables required to track the player's movement during gameplay.

3. Step Counting Logic

Purpose

Monitors player movement and updates the step count based on the travelled distance.

4. Request Data Model

Purpose

Defines the JSON structure used to transfer gameplay statistics to the backend.

5. Automatic Data Synchronization

Purpose

Periodically triggers the data transmission process using a Unity Coroutine.

6. JSON Serialization & API Request

Purpose

Converts the collected statistics into JSON, sends them to the Laravel REST API via an HTTP POST request, and handles the server response.
